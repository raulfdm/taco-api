## Bem-vindo(a)!

Primeiramente, obrigado por você estar interessado(a) em contribuir para a TACO GraphQL API! Este documento contém diretrizes para ajudar você a se tornar um(a) contribuidor(a) efetivo(a).

## Como Contribuir

Você pode contribuir de várias maneiras:

- Reportando bugs
- Sugerindo melhorias
- Escrevendo ou melhorando a documentação
- Enviando pull requests

### Reportando Bugs

Antes de reportar um bug, por favor, verifique se ele já foi reportado. Se não, crie uma nova issue explicando:

- O comportamento esperado
- O comportamento observado
- Passos para reproduzir o bug

### Sugerindo Melhorias

Novas ideias são sempre bem-vindas. Abra uma discussão na aba [discussions no Github](https://github.com/raulfdm/taco-api/discussions) descrevendo sua sugestão e como ela pode melhorar o projeto.

### Documentação

A documentação é crucial para entender um projeto. Se você notar falta de documentação ou erros, sinta-se à vontade para corrigir ou adicionar.

### Enviando Pull Requests

Siga estes passos para enviar um pull request:

1. Faça um fork do projeto e crie uma nova branch para suas mudanças.
2. Faça suas alterações e escreva testes quando aplicável.
3. Siga as convenções de código do projeto.
4. Atualize a documentação se necessário.
5. Envie um pull request descrevendo as mudanças propostas.

## Processo de Revisão

Cada contribuição será revisada por mim. Eu posso pedir mudanças ou dar sugestões.

## Perguntas

Se tiver dúvidas, acesse a página das [perguntas frequentes](/faq). Caso sua dúvida não esteja lá, abra uma nova discussão na aba [discussions](https://github.com/raulfdm/taco-api/discussions).

Obrigado por contribuir com o projeto!
